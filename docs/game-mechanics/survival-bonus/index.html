Survival Bonus

⚠️ Work in progress

The Survival Bonus is a gameplay mechanic designed to both reward and challenge players for avoiding death. It is tied to the character's Survival Time — the total time the character has spent actively engaged in combat without dying. The bonus begins at 0% and increases with diminishing returns toward a maximum of 100%. If the character dies, their Survival Time is reset, and the Survival Bonus returns to 0% as a result.

This mechanic does not apply to non-player characters.

The formulas are:

EffectiveSurvivalTime = SurvivalTime * (1 + BONUS_SURVIVAL_REL * 0.001)

SurvivalMultiplier = EffectiveSurvivalTime * UBmean / ((EffectiveSurvivalTime + UBpivot) * 500)

SurvivalBonus = 100 * MIN(1, SurvivalMultiplier)

Where:

  • SurvivalTime: The total time in seconds. It's not shown in-game, but it can be found in the character's stats file under the label alivetime. This value is already in seconds, so use it as-is, including the decimal part.
  • BONUS_SURVIVAL_REL: The total value for this bonus type (Faster Increase of Survival Bonus +X%).
  • UBmean: A parameter from balance.txt. The default value is 500 in the unmodified game.
  • UBpivot: A parameter from balance.txt. The default value is 21600 in the unmodified game. Setting it to 0 is equivalent to setting it to 1.
  • SurvivalBonus: The final Survival Bonus, expressed as a percentage.

To disable the Survival Bonus, set UBmean to 0.

The Survival Bonus has two known effects:

  • It grants a bonus to the character's attributes.
  • It increases the level of opponents.

While it is commonly believed that the Survival Bonus influences loot quality, there is no evidence to support this.

Bonus to Attributes

The formula is:

AttributeMultiplier = SurvivalMultiplier * 1000 / UBquot_attr

AttributeValue = FLOOR((FLOOR(FLOOR(CREATURE_BONUS_STATS * (1 + CREATURE_BONUS_STATS_REL * 0.001)) * (1 - DebuffMultiplier)) + AttributePoints) * AttributeMultiplier)

Where:

  • UBquot_attr: A parameter from balance.txt. The default value is 3000 in the unmodified game.
  • CREATURE_BONUS_STATS: The total value for the bonus BONUS_STATS (Attribute +X). Include only the bonuses that are attached to the creature directly; ignore bonuses from items or combat arts.
  • CREATURE_BONUS_STATS_REL: The total value for the bonus BONUS_STATS_REL (Attribute +X%). Include only the bonuses that are attached to the creature directly; ignore bonuses from items or combat arts.
  • AttributeValue: The bonus to a given attribute, expressed as an absolute value.

To disable this effect, set UBquot_attr to 0.

Increased Level of Opponents

Creatures normally spawn at the same level as the player's character. However, their final level is clamped within a level range specific to each creature instance.

  • Two otherwise identical creatures can spawn at different levels due to their individual level ranges.

The Survival Bonus increases the character's effective level for the purpose of determining creature spawn levels. The resulting creature level is still clamped to the creature's allowed range.

The formula is:

EffectiveLevel = CharacterLevel + FLOOR((CharacterLevel + 9) * SurvivalMultiplier * UBspawn_fact / 1000)

Where:

  • UBspawn_fact: A parameter from balance.txt. The default value is 175 in the unmodified game .

To disable this effect, set UBspawn_fact to 0.
